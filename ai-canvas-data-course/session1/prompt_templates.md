# Session 1: 프롬프트 템플릿 모음
## 복사-붙여넣기해서 바로 사용하세요!

**사용법:** 아래 프롬프트를 복사해서 AI 캔버스에 붙여넣고, `[대괄호]` 부분만 수정하세요.

## 📊 기본 데이터 분석 프롬프트

### 1. 데이터 첫 탐색

```
[파일명].csv 파일을 첨부했어.

이 데이터에 대해 알려줘:
1. 총 몇 개의 행과 열이 있는지
2. 각 컬럼의 이름과 데이터 타입
3. 결측값이 있는지
4. 처음 10개 행을 보여줘
5. 각 컬럼이 무엇을 의미하는지 추측해서 설명해줘
```

### 2. 기술 통계 요약

```
이 데이터의 기술 통계를 분석해줘:

수치형 변수:
- 평균, 중앙값, 표준편차
- 최소값, 최대값
- 사분위수

범주형 변수:
- 고유값 개수
- 각 값의 빈도

결과를 표로 정리해서 보여줘.
```

### 3. 데이터 품질 확인

```
데이터 품질을 점검해줘:

1. 결측값:
   - 어느 컬럼에 몇 개?
   - 비율은?
   - 시각화 (히트맵)

2. 이상치:
   - 박스플롯으로 확인
   - IQR 방법으로 탐지
   - 몇 개나 있는지

3. 중복값:
   - 완전히 동일한 행이 있는지

4. 데이터 타입 문제:
   - 숫자여야 하는데 문자로 저장된 것은 없는지

문제가 있으면 해결 방법도 제안해줘.
```

## 📈 시각화 프롬프트

### 4. 플랫폼별 비교 (막대그래프)

```
[platform] 컬럼을 기준으로 [likes] 컬럼의 평균을 계산하고,
막대그래프로 시각화해줘.

차트 스타일:
- 제목: "플랫폼별 평균 좋아요 수"
- 색상: 파란색 계열 (#2E86AB)
- 각 막대 위에 정확한 수치 표시
- 내림차순 정렬
- 배경: 흰색
- 그리드: 가로선만 연한 회색

전문적인 발표 자료에 쓸 거야!
```

### 5. 분포 확인 (히스토그램)

```
[likes] 컬럼의 분포를 히스토그램으로 보여줘.

포함사항:
- 적절한 bin 수 자동 선택
- 평균선 빨간 점선으로 표시
- 중앙값선 파란 점선으로 표시
- 정규분포 곡선 오버레이 (선택사항)
- 제목: "[변수명] 분포"

분포의 특징(정규분포인지, 치우침 등)도 설명해줘.
```

### 6. 관계 분석 (산점도)

```
[likes]와 [shares]의 관계를 산점도로 시각화해줘.

포함사항:
- 회귀선 추가 (빨간색 실선)
- 상관계수 표시
- 플랫폼별로 다른 색상으로 구분
- 범례 추가
- 제목: "좋아요 vs 공유 관계"

상관관계가 강한지 약한지도 해석해줘.
```

### 7. 시간 트렌드 (선 그래프)

```
[post_hour] 시간대별 평균 [likes]를 선 그래프로 보여줘.

스타일:
- 선 두께: 2px
- 마커: 동그라미
- 색상: #3498DB
- X축: 0시~23시
- Y축: 평균 좋아요
- 제목: "시간대별 참여도 트렌드"
- 피크 시간대에 주석 추가

언제 포스트하면 좋은지 인사이트도 줘!
```

## 🎯 그룹별 비교 프롬프트

### 8. 연령대별 패턴

```
[age_group]별로 [platform] 선호도를 분석해줘.

시각화:
1. 연령대-플랫폼 크로스탭 히트맵
2. 각 연령대에서 가장 인기있는 플랫폼 막대그래프

인사이트:
- 각 연령대의 특징
- 연령대별 타겟 전략 추천
```

### 9. 다중 그룹 비교

```
[platform]과 [age_group] 두 변수로 [likes]를 비교해줘.

시각화:
- 그룹 막대그래프 (grouped bar chart)
- 플랫폼을 X축, 연령대를 색상으로 구분
- 범례 우측 상단

분석:
- 어떤 조합이 가장 높은 참여도?
- 플랫폼별로 연령대 차이가 큰가?
- ANOVA 검정으로 통계적 유의성 확인
```

## 🔍 심화 분석 프롬프트

### 10. 상관관계 히트맵

```
[likes, shares, comments, time_spent] 변수들의
상관관계를 히트맵으로 시각화해줘.

스타일:
- 색상: RdBu_r (빨강-파랑)
- 각 셀에 상관계수 표시 (소수점 2자리)
- 대각선은 1로 표시
- 색상 범위: -1 ~ 1

해석:
- 가장 강한 양의 상관관계는?
- 가장 강한 음의 상관관계는?
- 의외의 발견은?
```

### 11. 종합 대시보드

```

## 🎯 프롬프트 작성 팁

### ✅ 좋은 프롬프트의 특징

1. **구체적:** "차트 만들어줘" ❌ → "파란색 막대그래프, 제목 포함" ✅
2. **맥락 제공:** "나는 커뮤니케이션 전공이고, 발표 자료용이야"
3. **예시 포함:** "이런 스타일로: [이미지/링크]"
4. **단계적:** 복잡한 작업은 여러 단계로 나누기
5. **확인 요청:** "맞는지 확인해줘", "다른 해석은 없을까?"

### ❌ 피해야 할 프롬프트

- "분석해줘" (너무 막연함)
- "안 돼" (무엇이 문제인지 설명 없음)
- "더 좋게" (어떤 방향으로?)
- "다시" (무엇을 바꿀지 명시 없음)

### 🔄 반복 개선 패턴

```
1차: "플랫폼별 평균 좋아요를 막대그래프로"
    → 기본 차트 생성

2차: "색상을 파란색으로, 제목 추가"
    → 스타일 개선

3차: "각 막대 위에 수치 표시"
    → 정보 추가

4차: "Plotly 인터랙티브로 변환"
    → 고급 기능 추가
```

## 📥 자주 묻는 질문

**Q: 프롬프트가 너무 길면 안 좋은가요?**
A: 아니요! 자세할수록 좋아요. AI는 긴 프롬프트를 잘 처리합니다.

**Q: 여러 번 나눠서 물어보는 게 나은가요?**
A: 관련된 작업은 한 번에 요청하는 게 좋아요. 맥락이 유지됩니다.

**Q: 프롬프트를 영어로 써야 하나요?**
A: 한국어로 편하게 쓰세요! Claude와 ChatGPT는 한국어를 잘 이해합니다.

**Q: 같은 프롬프트인데 결과가 다른데요?**
A: AI는 확률적이라 매번 조금씩 달라요. 더 나은 결과를 선택하세요!
